Auto-generated by `scripts/orchestration-linter.py`. Do not edit manually.

## Agents

- **yoink:decomp-evaluator**(library_name, package_name) → decision, reasoning, category, strategy, functions_to_replace, reference_material, acceptable_sub_dependencies: Evaluate whether a dependency should be kept or decomposed. Use during dependency decomposition (phase 3) to assess each library in the queue.
- **yoink:decomp-implementer**(top_package, sub_package, category, strategy, functions_to_replace, reference_material, acceptable_sub_dependencies, max_iterations) → completion_promise, what_was_done, new_imports: Implement or replace a dependency in yoink_<package>/ based on a decomposition evaluation. Use during dependency decomposition (phase 3) after evaluation.
- **yoink:test-discoverer**(package_name, target_function) → files_found, files_copied, summary: Search reference test suite for tests relevant to a target function. Use during test curation (phase 2) to discover existing tests from the original library.
- **yoink:test-generator**(package_name, target_function) → tests_written, test_files, summary: Generate focused pytest tests for a target function by studying reference implementation. Use during test curation (phase 2) after test discovery.
